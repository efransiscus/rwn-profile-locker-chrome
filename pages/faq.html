FAQ

What happens if I forget my PIN?
There is no recovery. You must remove the extension from chrome://extensions and load it again. All data is wiped and you will set a new PIN.
Why did all my tabs go away when I unlocked?
They didn't. The extension restores your previous tabs when you unlock. If a tab was opened while locked, it has no previous page to return to, so it falls back to a new tab.
Does this work in Chrome Guest mode?
No — and that is intentional. Guest mode is your "agent-safe" browsing space. The extension is not installed there, so agents running in Guest mode are never blocked by it.
Can I change my PIN?
Yes. Open the extension options (from the popup or right-click the icon → Options) and use the Change PIN section.
How do I completely remove the lock?
Go to chrome://extensions, find rwn Profile Lock, and click Remove. This deletes the PIN hash and all settings.
Is my PIN stored securely?
Your raw PIN is never stored. Only a PBKDF2-SHA256 hash (150,000 iterations, random 16-byte salt) is kept in local storage. The salt and hash are visible to anyone with DevTools access — this is an accepted limitation of browser-extension security.
Why does the screen flash for a split second on startup?
Chrome restores your session tabs before the extension service worker wakes up. There is a brief (~500ms) window where restored tabs are visible before they are redirected to the lock page. This is a known Chrome limitation.
Will this stop AI agents completely?
No. This is a simple guardrail, not a hardened security boundary. Agents with OS-level access, DevTools Protocol, or the ability to disable extensions can bypass it. Run agents in Chrome's Guest profile for proper isolation.
